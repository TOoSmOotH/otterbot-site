Getting Started

Get OtterBot up and running in minutes. Docker is the recommended approach — one command and you're live.

On this page
Prerequisites Quick Start with Docker Development Setup First-Run Setup Wizard Environment Variables

# Prerequisites

OtterBot has minimal requirements depending on how you run it:

Docker (recommended)

  • Docker 20.10+ (or Docker Desktop)
  • At least 2 GB RAM available for the container
  • An LLM API key (Anthropic, OpenAI, or any OpenAI-compatible provider)

From Source

  • Node.js 20+ and npm 10+
  • Git
  • An LLM API key
  • Optional: Playwright browsers (for web automation features)

# Quick Start with Docker

Step 01 — Pull the image
$ docker pull ghcr.io/toosmooth/otterbot:latest
Step 02 — Run the container
$ docker run -d -p 8080:8080 \
  --name otterbot \
  ghcr.io/toosmooth/otterbot:latest
Step 03 — Open the UI

Navigate to http://localhost:8080 in your browser. The first-run setup wizard will guide you through configuring your LLM provider and API key.

🌱

Persist data between sessions: OtterBot is ephemeral by default. Mount a volume to keep your data: -v ~/otterbot-data:/data

Full Docker command with all options

$ docker run -d \
  --name otterbot \
  -p 8080:8080                # Web UI
  -v ~/otterbot-data:/data      # Persistent storage
  -e ENABLE_DESKTOP=true      # Enable virtual desktop
  ghcr.io/toosmooth/otterbot:latest

# Development Setup

To run OtterBot from source for development or contribution:

# Clone the repository
$ git clone https://github.com/TOoSmOotH/otterbot.git
$ cd otterbot

# Install dependencies (monorepo — installs all packages)
$ npm install

# Build the shared types package first
$ npm run build --workspace packages/shared

# Start the dev server (server + web concurrently)
$ npm run dev

The development server runs on http://localhost:62626 by default. The web frontend is built with React + Vite and hot-reloads on changes.

📚

Monorepo structure: OtterBot uses npm workspaces with three packages: @otterbot/server, @otterbot/web, and @otterbot/shared. See Architecture for details.

# First-Run Setup Wizard

When you first open OtterBot, the setup wizard walks you through the essential configuration:

1. LLM Provider

Choose your AI provider and enter your API key. Supported providers:

  • Anthropic — Claude models (recommended)
  • OpenAI — GPT models
  • OpenAI-Compatible — Any provider with an OpenAI-compatible API (Ollama, LM Studio, etc.)

The wizard probes available models from your provider so you can select which model to use for the COO agent and worker agents.

2. Voice Setup (optional)

Configure text-to-speech and speech-to-text if desired. You can preview voices during setup. See Features → Voice for provider details.

3. Search Provider (optional)

Set up web search for your agents. DuckDuckGo works out of the box with no API key. See Features → Web Search for all providers.
